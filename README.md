# 我的 AI / Agent 工程知识体系

这是我的 AI / Agent 工程完整知识体系。目标是系统性地梳理这个领域的概念、技术、工具和演进脉络。

## 怎么用这个项目

### 配合 Claude Code 的工作流

打开 Claude Code，在这个目录下开始对话。Claude 会自动读取 `CLAUDE.md` 来理解协作协议。

四种典型对话开场：

| 我说什么 | Claude 进入什么模式 |
|---|---|
| "我觉得 X..." / "你觉得 Y 对吗" | 对话伙伴模式（推演 / 反驳） |
| "归档这个" / 贴一个链接 | 归档员模式（归位 + 写入文件） |
| "帮我搞清楚 X" / "X 的现状" | 研究员模式（搜索 + 结构化简报） |
| "检查体系" / "哪里需要更新" | 体检员模式（诊断 + 报告） |

### 自己看的时候

- 想看全貌：从 `overview.md` 开始
- 想看某个维度：去 `dimensions/`
- 想看演进：去 `timeline/`
- 想看最近发生了什么：去 `logs/collapse-emergence-log.md`

## 体系的核心思想

详见 `CLAUDE.md` 第 2 节和 `overview.md`。

简言之：
- **空间索引**：地基-建筑结构。LLM 是地基（七维度），Agent 工程是建筑（四层）
- **时间索引**：代际演进。不按年月，按地基质变划代
- **核心机制**：地基变化 → 建筑塌缩或涌现
- **核心动作**：每个新进展都被这两个索引同时锚定

## 维护原则

- 先全景再追踪：先建立完整知识地图，再在此基础上追踪变化
- 空白是待填的：覆盖缺失应被识别和补上
- 双索引锚定：任何条目都要能在维度图和时间线上定位
- 全景 + 变化：既描述"当前是什么样"，也追踪"最近发生了什么变化"
